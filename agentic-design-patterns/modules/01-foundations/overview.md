# Module 01 — Foundations

**Days 1–5 · 4 pages + 1 rest day**

---

## What this module earns you

You arrive in this module knowing what an agent *does*. You leave knowing what an agent *is* — precisely, structurally, in terms that transfer to every paper, every system, and every pattern discussion you will have for the rest of the course.

Concretely, by the end of Day 5 you can:

1. Define an agentic design pattern and explain why the naming matters (Day 1)
2. Decompose any agent system into four orthogonal components using the cognitive architecture map (Day 2)
3. Describe the complete action space taxonomy and explain why tools are a first-class primitive (Day 3)
4. Read any agent execution trace and identify which components are active at each step (Day 4)

The rest day (Day 5) is when these four ideas fuse into a single stable mental model you can use without effort.

---

## The through-line

The module answers one question: *What is the thing we are designing patterns for?*

Before you can recognize a pattern, you need a vocabulary for the system the pattern lives in. A doctor who doesn't know anatomy can't diagnose. An architect who doesn't know load-bearing walls can't read a blueprint. This module is anatomy class — short, necessary, and the foundation everything else stands on.
