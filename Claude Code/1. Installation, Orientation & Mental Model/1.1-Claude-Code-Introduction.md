# Claude Code: Agentic Coding, Installation & First Run
**Last Updated:** 2026-04-15

> *Your coding assistant just grew up — it doesn't complete lines anymore, it completes tasks. Here's what that actually means and how to get started in 10 minutes.*

---

## Learning Outcomes

By the end of this page, you will be able to:

- Explain, in plain terms, why Claude Code is fundamentally different from autocomplete tools like GitHub Copilot
- Describe what "agentic coding" means and why the distinction matters for how you use the tool
- Install Claude Code as a CLI tool and as a VS Code extension on your machine
- Launch Claude Code on a real existing project and run your first meaningful task

## 1. Why This Matters (In Our Systems)

Imagine you've just joined a codebase with 80,000 lines you've never touched. Someone asks you to add rate limiting to the API layer. With a regular code assistant, you'd get tab-completions as you type — helpful, but you're still the one navigating files, reading context, writing logic, and wiring things together.

Now imagine a colleague sits down next to you. You say *"add rate limiting to the API layer"* and they open files, read the existing middleware, write the implementation, hook it into the router, and show you a diff to review.

That's the shift. Most teams adopting AI tools stay stuck in the "better autocomplete" mental model long after better tools exist. The engineers who understand the agentic model get 3–10x more leverage from the same hours.

Not understanding this difference means using a power drill to stir paint.

## 2. Intuition & Mental Models

Think about how autocomplete works. You type `const user = await db.get` and the tool predicts what comes next — it's completing your thought, one token at a time, with no memory of what you were about to do next.

Now think about how you delegate a task to a junior engineer. You don't narrate every keystroke. You hand them a goal, they figure out the steps, they come back with something reviewable.

**Autocomplete is word prediction. Agentic coding is task delegation.**

Claude Code operates in a loop:
1. You give it a goal in natural language
2. It reads your codebase to understand context
3. It plans a sequence of actions (read file → edit file → run tests → check output)
4. It executes those actions with real tools (file reads/writes, terminal commands, search)
5. It loops until the goal is met or it needs your input

The key insight: Claude Code has *tools* — it can actually run `grep`, read files, execute shell commands, and check results. It's not guessing what to write next. It's reasoning about what to *do* next.

---

## 3. Core Concepts & Terminology

**Agentic coding** — A coding workflow where an AI model autonomously plans and executes multi-step tasks using tools (file I/O, terminal, search), rather than passively suggesting completions.

**Autocomplete / Inline suggestions** — The "ghost text" model: you type, the model predicts the next token(s). GitHub Copilot's core feature. Fast, low-commitment, context is limited to what's visible in the editor.

**Context window** — The amount of code + conversation history Claude can "see" at once. Claude Code actively manages this by reading only relevant files, rather than dumping your entire repo in.

**Tool use** — Claude Code's ability to call real functions: read a file, write to a file, run a bash command, search the codebase. This is what separates it from chat-based assistants.

**Permission model** — Before Claude Code takes actions that could be irreversible (deleting files, running commands), it asks. You control how much autonomy it has.

**CLAUDE.md** — A special file you can place in your repo root. Claude Code reads it automatically and uses it as persistent context about your project (architecture decisions, coding conventions, off-limits patterns). Think of it as onboarding docs for your AI collaborator.

## 4. How It Works (What Actually Matters)

When you run a task, Claude Code enters an agent loop:

```
Goal received
    → Plan (what files are relevant? what steps are needed?)
    → Act (read file, edit code, run terminal command)
    → Observe (did the test pass? did the build succeed?)
    → Repeat until done or blocked
```

It uses the tools available to it — file reads, writes, bash execution — to gather information and make changes. Each action is shown to you in the terminal so you can follow along and interrupt at any point.

> ⚠️ **Counterintuitive:** Claude Code doesn't load your entire codebase upfront. It reads files *on demand* as it reasons about the task. This means it can work on large repos without context overflow — but it also means it can miss things that aren't obviously discoverable from the entry point you give it.

The VS Code extension adds a sidebar panel that mirrors the terminal session and lets you view diffs inline. The underlying model and behavior are identical — it's the same agent, different surface.

## 5. Worked Examples & Realistic Scenarios

**Scenario:** You have a Node.js Express app. You want to add request logging middleware.

Without Claude Code, you'd open `app.js`, find where middleware is registered, write the logger, wire it in, maybe check if a logging library is already installed.

With Claude Code, you open your terminal in the project root and type:

```bash
claude
```

Then in the Claude Code prompt:

```
Add request logging middleware to this Express app. Log method, path, status code,
and response time. Use morgan if it's already a dependency; otherwise use a simple
custom implementation.
```

Claude Code will:
1. Read `package.json` to check for `morgan`
2. Read `app.js` (or wherever middleware is registered) to understand the structure
3. Write the implementation — using `morgan` if found, or a clean custom logger if not
4. Show you the diff and ask for confirmation before writing

You reviewed the plan, you saw the diff, you approved. The task took 40 seconds instead of 10 minutes. More importantly, it made the *right* decision based on what was actually in your repo.

## 6. Practical Usage & Decision Guidance

**Reach for Claude Code when:**
- The task requires understanding existing code before writing new code
- You need to make changes across multiple files
- You're in an unfamiliar codebase and need to move fast
- You want to automate something you'd otherwise do manually (e.g., "rename this function everywhere it's used")

**Reach for Copilot / inline autocomplete when:**
- You're in flow and just want fast completions as you type
- The task is purely generative with no need to read existing context
- You want suggestions without committing to a full task loop

**Decision table:**

| Situation | Better tool |
|---|---|
| "What does this function do?" | Either (Claude Code gives more depth) |
| "Add a field to this form + update validation" | Claude Code |
| "Complete this function signature I started" | Copilot / autocomplete |
| "Refactor this module to use async/await" | Claude Code |
| "Write a boilerplate React component" | Either |
| "Fix the bug in this error log" | Claude Code (it can read the files) |

## 7. Common Pitfalls & Misconceptions

**"It'll just do it all for me."**
Most people assume Claude Code is autonomous in the "set it and forget it" sense. In reality, it works best as a *collaborative* loop — you give goals, review diffs, steer when it goes wrong. Treating it like a black box leads to merged code you don't understand.

**"I'll give it the whole feature spec at once."**
The reality: shorter, focused tasks produce better results than large vague goals. "Refactor the entire authentication system" will go worse than "Extract the token validation logic into its own module." Decompose first.

**"It knows my codebase."**
Claude Code reads what it needs, but it starts fresh each session (unless you have a `CLAUDE.md`). It doesn't remember decisions from last Tuesday. Invest 20 minutes writing a `CLAUDE.md` — it pays back every session.

**"It's just a fancier chat window."**
This trips up people coming from ChatGPT. Claude Code has real tool access. It actually runs commands. If you tell it to delete something, it can delete it. The permission prompts exist for a reason — read them.

---

## 8. Trade-offs, Scale, and Edge Cases

Claude Code shines on tasks with clear success criteria and good test coverage. The agent can run your tests and verify its own work. Without tests, you're back to manual verification — which is fine, but understand you've removed one of its feedback loops.

On very large monorepos, file discovery can be slow and the agent may not find the right entry points without guidance. The `CLAUDE.md` file is your lever here — point it at the right directories and conventions explicitly.

The VS Code extension is convenient for reviewing diffs in-editor, but the CLI is more powerful for scripting, CI integration, and running headless tasks. Don't assume the extension is the "right" way — many heavy users live in the terminal.

> ⚠️ **Counterintuitive:** Claude Code with broad file-write permissions in a CI pipeline is a powerful pattern (automated PR generation, automated refactors at scale), but it requires the same review discipline you'd apply to any automated code change. Treat its output like a junior engineer's PR — capable, but not rubber-stamp it.

3. What is `CLAUDE.md` and why should it be the first file you create when onboarding Claude Code to an existing project?

## 10. What to Learn Next

- **CLAUDE.md & Project Configuration** — Learning to write a great `CLAUDE.md` is the single highest-leverage thing you can do to improve result quality on your specific codebase.
- **Hooks & Automation** — Claude Code supports pre/post-tool hooks, letting you enforce conventions (run linter after every edit, block writes to certain paths). This is where it scales from personal tool to team practice.
- **Multi-Agent Workflows** — Claude Code can spawn subagents for parallel tasks. Once you've mastered single-agent use, understanding how to decompose larger tasks across agents is the next ceiling to break.
- **Prompt Engineering for Agentic Tasks** — Writing effective goals for an agent is a different skill from writing effective chat prompts. Specificity, success criteria, and constraints matter more here.

## Installation

### Prerequisites
- Node.js 18 or higher (`node --version` to check)
- An Anthropic API key, or a Claude.ai Pro/Team/Enterprise subscription
- **Windows users:** Git Bash (or WSL) — Claude Code expects a POSIX-style shell, and PowerShell/CMD aren't supported. macOS and Linux users can skip the Git Bash section below.

### Windows: Install Git Bash
Git Bash ships as part of **Git for Windows** and gives you the Bash shell, common Unix utilities (`ls`, `grep`, `cat`, etc.), and proper handling of POSIX-style paths — all of which Claude Code relies on.

1. Download the installer from [[git-scm.com/download/win](https://git-scm.com/download/win)](https://git-scm.com/download/win). It should start automatically; if not, pick the 64-bit standalone installer.
2. Run the installer. The defaults are fine for almost everyone, but pay attention to these screens:
   - **Select Components:** keep "Git Bash Here" and "Git GUI Here" checked — the right-click menu entries are the easiest way to open a shell in a project folder.
   - **Choosing the default editor:** pick whatever you actually use (VS Code is a good default).
   - **Adjusting your PATH environment:** select **"Git from the command line and also from 3rd-party software"** (the recommended middle option). This lets `git` work from PowerShell/CMD too, without overwriting your system tools.
   - **Configuring the terminal emulator:** keep the default **MinTTY** — it's what `Git Bash` actually launches into.
   - **Configuring extra options:** leave "Enable file system caching" checked.
3. Finish the installer and launch **Git Bash** from the Start menu (or right-click any folder in Explorer and choose **"Git Bash Here"**).
4. Verify it works:
   ```bash
   bash --version
   git --version
   node --version
   ```
   If `node` isn't found, install Node.js from [[nodejs.org](https://nodejs.org/)](https://nodejs.org/) using the Windows installer, then close and reopen Git Bash.

From here on, **run every command in this guide inside a Git Bash window**, not PowerShell or CMD.

### CLI Installation
```bash
npm install -g @anthropic-ai/claude-code
```

Verify it installed:
```bash
claude --version
```

Authenticate (first run will prompt you, or set manually):
```bash
export ANTHROPIC_API_KEY="sk-ant-..."
```

For persistence, add that export to your `~/.bashrc`, `~/.zshrc`, or equivalent. On Windows + Git Bash, the file is `~/.bashrc` inside your Git Bash home (which maps to `C:\Users\<you>`). If `~/.bashrc` doesn't exist yet, create it — Git Bash will source it on the next launch.

### VS Code Extension
1. Open VS Code
2. Press `Ctrl+Shift+X` (Extensions panel)
3. Search for **Claude Code**
4. Install the extension published by Anthropic
5. Reload VS Code when prompted

The extension connects to the same CLI under the hood. If the CLI is installed and authenticated, the extension works automatically. You'll see a Claude Code panel in the sidebar (Activity Bar).

**Windows tip:** make sure VS Code's integrated terminal is set to Git Bash (Terminal → Select Default Profile → **Git Bash**). The extension and its terminal commands will then behave the same way they do on macOS/Linux.

### Your First Run on a Real Project
Navigate to an existing project you know well — something small, with a few files:
```bash
cd path/to/your/project
claude
```

You're now in the Claude Code interactive session. Try this as your first task:
```
Read the main entry point of this project and give me a 3-sentence summary
of what it does and how it's structured.
```

This is a read-only task — safe, low stakes, and immediately useful. It also shows you what Claude Code "sees" when it enters your project. From here, try a small edit:
```
Add a comment block at the top of the main file explaining what this
module does and what its main dependencies are.
```

Review the diff it shows you before confirming. That review habit is the foundation of working well with agentic tools.

## Summary

Claude Code is an agentic coding tool, not an autocomplete engine. The difference is structural: it uses real tools (file reads, writes, terminal commands) to plan and execute multi-step tasks, rather than predicting the next token as you type. Installation takes under 5 minutes via npm or the VS Code extension. The fastest path to understanding it is running it on a codebase you already know — start with a read-only task, build trust with the diff review loop, and invest early in a `CLAUDE.md` file to give it persistent context about your project.
