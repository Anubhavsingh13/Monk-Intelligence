## 1. Overview & Architecture Philosophy

### What Is an AI Agent?

An AI Agent is a system where an LLM **reasons, plans, and takes actions** — iteratively — to accomplish a goal. Unlike a single prompt→response flow, agents:

- Decompose complex tasks into sub-steps
- Choose from a set of **tools** (APIs, databases, code executors)
- Observe results and **re-plan** if needed
- Maintain **memory** across turns (short-term) and runs (long-term)

### System Blueprint

```
┌─────────────────────────────────────────────────────────┐
│                    Azure Function (HTTP Trigger)         │
│  ┌──────────────────────────────────────────────────┐   │
│  │              LangGraph Agent Graph                │   │
│  │                                                   │   │
│  │  [User Input] → [Router Node]                    │   │
│  │                      │                            │   │
│  │         ┌────────────┼────────────┐               │   │
│  │         ▼            ▼            ▼               │   │
│  │  [Reasoning]  [Customer Support]  [Escalate]     │   │
│  │         │            │                            │   │
│  │         └────────────┘                            │   │
│  │                  │                                │   │
│  │            [Tool Executor]                        │   │
│  │                  │                                │   │
│  │       [Azure OpenAI GPT-4o]                       │   │
│  └──────────────────────────────────────────────────┘   │
│                                                          │
│  External Tools: Search · CRM · Knowledge Base · SQL    │
└─────────────────────────────────────────────────────────┘
```

### Two Agents We Build

| Agent | Purpose | Reasoning Style |
|---|---|---|
| **Multi-Step Reasoning Agent** | Answers complex queries by breaking them into sub-tasks, using tools | ReAct (Reason + Act) loop |
| **Customer Support Agent** | Handles support tickets, escalates, updates CRM | Goal-directed dialogue with guardrails |

```python
from langchain_core.tools import tool


@tool
def calculate(expression: str) -> str:
    """
    Evaluate a mathematical expression safely.

    Use this for arithmetic, algebra, unit conversions,
    or any numerical computation.

    Args:
        expression: A Python-safe math expression (e.g., "2 ** 10 + 42")

    Returns:
        The computed result as a string
    """
    import ast
    import math
    import operator

    # Safe evaluation — only allow math operations
    allowed_names = {
        k: v for k, v in math.__dict__.items() if not k.startswith("__")
    }
    allowed_names.update({"abs": abs, "round": round})

    try:
        tree = ast.parse(expression, mode="eval")
        result = eval(compile(tree, "<string>", "eval"), {"__builtins__": {}}, allowed_names)
        return str(result)
    except Exception as e:
        return f"Calculation error: {e}"
```

### Step 5.3 — LangGraph State & Graph

```python
# src/agents/reasoning/graph.py
from typing import Annotated, Sequence
from langchain_core.messages import BaseMessage, HumanMessage, AIMessage
from langgraph.graph import StateGraph, END
from langgraph.graph.message import add_messages
from langgraph.prebuilt import ToolNode
from pydantic import BaseModel


class ReasoningState(BaseModel):
    """State schema for the reasoning agent graph."""

    messages: Annotated[Sequence[BaseMessage], add_messages]
    iteration_count: int = 0
    final_answer: str | None = None


def build_reasoning_graph(llm_with_tools, tools: list, max_iterations: int = 10):
    """
    Build a LangGraph ReAct-style reasoning graph.

    Graph structure:
        START → agent_node → (tool_node | END)
                    ↑              |
                    └──────────────┘

    Args:
        llm_with_tools: LLM instance bound with tools via .bind_tools()
        tools: List of LangChain tool objects
        max_iterations: Safety limit to prevent infinite loops

    Returns:
        Compiled LangGraph runnable
    """

    def agent_node(state: ReasoningState) -> dict:
        """Core reasoning node — calls LLM and decides next action."""
        if state.iteration_count >= max_iterations:
            return {
                "final_answer": "Maximum reasoning steps reached. Providing best available answer.",
                "messages": state.messages,
            }

        response = llm_with_tools.invoke(state.messages)
        return {
            "messages": [response],
            "iteration_count": state.iteration_count + 1,
        }

    def should_continue(state: ReasoningState) -> str:
        """
        Router: decide whether to call tools or end.

        Returns:
            "tools"  — LLM wants to call a tool
            "end"    — LLM produced a final answer
        """
        last_message = state.messages[-1]

        # If the LLM made tool calls, route to tool executor
        if hasattr(last_message, "tool_calls") and last_message.tool_calls:
            return "tools"

        # Otherwise, we have a final answer
        return "end"

    # Build the graph
    workflow = StateGraph(ReasoningState)

    # Add nodes
    workflow.add_node("agent", agent_node)
    workflow.add_node("tools", ToolNode(tools))

    # Define edges
    workflow.set_entry_point("agent")
    workflow.add_conditional_edges(
        "agent",
        should_continue,
        {"tools": "tools", "end": END},
    )
    workflow.add_edge("tools", "agent")  # After tool execution → back to agent

    return workflow.compile()
```

### Step 5.4 — Agent Class

```python
# src/agents/reasoning/agent.py
from langchain_core.messages import HumanMessage, SystemMessage
from langchain_core.runnables import RunnableConfig
from .graph import ReasoningState, build_reasoning_graph
from .prompts import REASONING_SYSTEM_PROMPT
from ..llm import get_azure_llm
from ..tools.search import web_search
from ..tools.calculator import calculate
from ..tools.knowledge_base import query_knowledge_base


class MultiStepReasoningAgent:
    """
    A multi-step reasoning agent powered by Azure OpenAI + LangGraph.

    The agent uses a ReAct loop: it reasons about the problem,
    selects and executes tools, observes results, and repeats
    until it reaches a final answer.

    Usage:
        agent = MultiStepReasoningAgent()
        result = await agent.run("What is the GDP of France divided by its population?")
    """

    def __init__(self, max_iterations: int = 10):
        self.tools = [web_search, calculate, query_knowledge_base]
        self.llm = get_azure_llm(temperature=0.0)
        self.llm_with_tools = self.llm.bind_tools(self.tools)
        self.graph = build_reasoning_graph(
            self.llm_with_tools,
            self.tools,
            max_iterations=max_iterations,
        )

    async def run(self, user_message: str, thread_id: str = "default") -> dict:
        """
        Run the reasoning agent on a user query.

        Args:
            user_message: The question or task from the user
            thread_id: Unique ID for conversation memory (checkpointing)

        Returns:
            dict with keys: answer (str), steps (list), usage (dict)
        """
        initial_state = ReasoningState(
            messages=[
                SystemMessage(content=REASONING_SYSTEM_PROMPT),
                HumanMessage(content=user_message),
            ]
        )

        config = RunnableConfig(
            configurable={"thread_id": thread_id},
            tags=["reasoning-agent"],
        )

        final_state = await self.graph.ainvoke(initial_state, config=config)

        # Extract the final AI response
        ai_messages = [
            m for m in final_state["messages"]
            if isinstance(m, type(final_state["messages"][-1]))
            and not getattr(m, "tool_calls", None)
        ]

        return {
            "answer": final_state["messages"][-1].content,
            "steps": len(final_state["messages"]),
            "iterations": final_state.get("iteration_count", 0),
        }
```

## 6. Building the Customer Support Agent

This agent handles customer tickets with a **goal-directed dialogue graph** — routing issues, applying guardrails, querying the knowledge base, and escalating when needed.

### Step 6.1 — Support State Schema

```python
# src/agents/support/graph.py
from typing import Annotated, Literal
from pydantic import BaseModel, Field
from langchain_core.messages import BaseMessage
from langgraph.graph.message import add_messages


class SupportTicket(BaseModel):
    """Incoming support request schema."""
    ticket_id: str
    customer_id: str
    message: str
    channel: Literal["email", "chat", "phone"] = "chat"
    priority: Literal["low", "medium", "high", "critical"] = "medium"


class SupportState(BaseModel):
    """Full state for customer support dialogue graph."""

    # Conversation history
    messages: Annotated[list[BaseMessage], add_messages]

    # Ticket metadata
    ticket: SupportTicket | None = None

    # Agent decisions
    intent: str = ""                   # Classified intent
    resolved: bool = False             # Was issue resolved?
    escalated: bool = False            # Was ticket escalated?
    escalation_reason: str = ""

    # Knowledge retrieved
    kb_results: list[str] = Field(default_factory=list)

    # CRM actions taken
    crm_updated: bool = False
    resolution_summary: str = ""
```

### Step 6.2 — Support Graph Nodes

```python
# src/agents/support/graph.py (continued)
from langchain_core.messages import SystemMessage, HumanMessage, AIMessage
from langgraph.graph import StateGraph, END
from ..llm import get_azure_llm
from ..tools.knowledge_base import query_knowledge_base
from ..tools.crm import update_crm_ticket
from .prompts import (
    INTENT_CLASSIFIER_PROMPT,
    SUPPORT_AGENT_PROMPT,
    ESCALATION_PROMPT,
)


def classify_intent_node(state: SupportState) -> dict:
    """
    Node: Classify the customer's intent.

    Categories: billing | technical | account | general | out_of_scope
    """
    llm = get_azure_llm(temperature=0.0)

    classification_prompt = INTENT_CLASSIFIER_PROMPT.format(
        message=state.ticket.message if state.ticket else state.messages[-1].content
    )

    response = llm.invoke([SystemMessage(content=classification_prompt)])
    intent = response.content.strip().lower()

    return {"intent": intent}


def retrieve_knowledge_node(state: SupportState) -> dict:
    """
    Node: Query knowledge base for relevant articles.
    """
    query = state.ticket.message if state.ticket else state.messages[-1].content
    result = query_knowledge_base.invoke({"question": query})

    return {"kb_results": [result]}


def agent_response_node(state: SupportState) -> dict:
    """
    Node: Generate support response using LLM + retrieved knowledge.
    """
    llm = get_azure_llm(temperature=0.2)

    kb_context = "\n\n".join(state.kb_results) if state.kb_results else "No relevant articles found."

    system_prompt = SUPPORT_AGENT_PROMPT.format(
        intent=state.intent,
        kb_context=kb_context,
        customer_id=state.ticket.customer_id if state.ticket else "unknown",
    )

    all_messages = [SystemMessage(content=system_prompt)] + list(state.messages)
    response = llm.invoke(all_messages)

    return {
        "messages": [response],
        "resolved": "resolved" in response.content.lower(),
    }


def escalation_node(state: SupportState) -> dict:
    """
    Node: Handle escalation to human agent.
    """
    escalation_summary = f"""
    ESCALATION TRIGGERED
    Ticket ID: {state.ticket.ticket_id if state.ticket else 'N/A'}
    Customer: {state.ticket.customer_id if state.ticket else 'N/A'}
    Intent: {state.intent}
    Priority: {state.ticket.priority if state.ticket else 'unknown'}
    Reason: Automated agent could not resolve — routing to human support team.
    """

    return {
        "escalated": True,
        "escalation_reason": escalation_summary,
        "messages": [AIMessage(content=(
            "I'm connecting you with a specialist who can better assist you. "
            "You'll hear back within 2 business hours. Ticket ID: "
            f"{state.ticket.ticket_id if state.ticket else 'N/A'}"
        ))],
    }


def route_after_response(state: SupportState) -> str:
    """
    Router: After generating a response, decide what happens next.

    Returns:
        "escalate" — LLM flagged issue needs human intervention
        "end"      — Issue resolved or response sent
    """
    if state.escalated:
        return "end"

    last_message = state.messages[-1]
    content = last_message.content.lower() if hasattr(last_message, "content") else ""

    # Escalation triggers
    escalation_keywords = ["escalat", "human agent", "specialist", "refund", "legal", "complaint"]
    if any(kw in content for kw in escalation_keywords):
        return "escalate"

    if state.intent == "out_of_scope":
        return "escalate"

    return "end"


def build_support_graph():
    """
    Build the customer support LangGraph.

    Graph flow:
        START
          │
          ▼
    [classify_intent]
          │
          ▼
    [retrieve_knowledge]
          │
          ▼
    [agent_response] ──── "escalate" ──► [escalation] ──► END
          │
          └──── "end" ──► END
    """
    workflow = StateGraph(SupportState)

    workflow.add_node("classify_intent", classify_intent_node)
    workflow.add_node("retrieve_knowledge", retrieve_knowledge_node)
    workflow.add_node("agent_response", agent_response_node)
    workflow.add_node("escalation", escalation_node)

    workflow.set_entry_point("classify_intent")
    workflow.add_edge("classify_intent", "retrieve_knowledge")
    workflow.add_edge("retrieve_knowledge", "agent_response")
    workflow.add_conditional_edges(
        "agent_response",
        route_after_response,
        {"escalate": "escalation", "end": END},
    )
    workflow.add_edge("escalation", END)

    return workflow.compile()
```

### Step 6.3 — Support Agent Class

```python
# src/agents/support/agent.py
from langchain_core.messages import HumanMessage
from langchain_core.runnables import RunnableConfig
from .graph import SupportState, SupportTicket, build_support_graph


class CustomerSupportAgent:
    """
    Customer support agent with intent classification, knowledge retrieval,
    and escalation routing.

    Usage:
        agent = CustomerSupportAgent()
        result = await agent.handle_ticket(ticket)
    """

    def __init__(self):
        self.graph = build_support_graph()

    async def handle_ticket(self, ticket: SupportTicket) -> dict:
        """
        Process an incoming support ticket end-to-end.

        Args:
            ticket: Validated SupportTicket model

        Returns:
            dict with: response, resolved, escalated, intent, ticket_id
        """
        initial_state = SupportState(
            messages=[HumanMessage(content=ticket.message)],
            ticket=ticket,
        )

        config = RunnableConfig(
            configurable={"thread_id": ticket.ticket_id},
            tags=["support-agent", f"priority:{ticket.priority}"],
        )

        final_state = await self.graph.ainvoke(initial_state, config=config)

        # Get the last AI response
        ai_responses = [
            m for m in final_state["messages"]
            if isinstance(m, HumanMessage) is False
        ]
        last_response = ai_responses[-1].content if ai_responses else ""

        return {
            "ticket_id": ticket.ticket_id,
            "response": last_response,
            "intent": final_state.get("intent", ""),
            "resolved": final_state.get("resolved", False),
            "escalated": final_state.get("escalated", False),
            "escalation_reason": final_state.get("escalation_reason", ""),
        }
```

## 7. LangGraph State Machines & Orchestration

### Key LangGraph Concepts

| Concept | Description | Example |
|---|---|---|
| `StateGraph` | The graph container — holds nodes and edges | `workflow = StateGraph(MyState)` |
| **Node** | A Python function that transforms state | `def my_node(state) → dict:` |
| **Edge** | Deterministic connection between nodes | `workflow.add_edge("a", "b")` |
| **Conditional Edge** | Router that picks next node dynamically | `workflow.add_conditional_edges(...)` |
| **Checkpointer** | Persists state between invocations | `SqliteSaver`, `PostgresSaver` |
| `END` | Terminal node — graph execution stops here | `workflow.add_edge("last_node", END)` |

### Adding Memory with Checkpointing


### Parallel Tool Execution (Advanced)

```python
# src/agents/reasoning/graph.py — parallel subgraph
from langgraph.graph import StateGraph, END
from typing import Annotated
import operator


class ParallelResearchState(BaseModel):
    query: str
    search_results: Annotated[list, operator.add] = []  # Parallel writes merge here
    kb_results: Annotated[list, operator.add] = []


def build_parallel_research_graph(llm, search_tool, kb_tool):
    """
    Execute web search and KB lookup in parallel, then synthesize.

    START
      ├──► [web_search_node]  ──► [synthesize]
      └──► [kb_lookup_node]   ──►

    Both branches run concurrently — LangGraph merges results automatically.
    """
    workflow = StateGraph(ParallelResearchState)

    workflow.add_node("web_search", lambda s: {"search_results": [search_tool(s.query)]})
    workflow.add_node("kb_lookup", lambda s: {"kb_results": [kb_tool(s.query)]})
    workflow.add_node("synthesize", lambda s: {
        "messages": [llm.invoke(f"Synthesize: {s.search_results} {s.kb_results}")]
    })

    workflow.set_entry_point("web_search")
    workflow.set_entry_point("kb_lookup")  # Parallel branch
    workflow.add_edge("web_search", "synthesize")
    workflow.add_edge("kb_lookup", "synthesize")
    workflow.add_edge("synthesize", END)

    return workflow.compile()
```

### Step 8.2 — Reasoning Agent Azure Function

```python
# functions/reasoning_agent/__init__.py
import azure.functions as func
import logging
import json
from pydantic import BaseModel, ValidationError

# Lazy import — agent initializes once (warm start optimization)
_agent = None

def get_agent():
    global _agent
    if _agent is None:
        from agents.reasoning.agent import MultiStepReasoningAgent
        _agent = MultiStepReasoningAgent()
    return _agent


class ReasoningRequest(BaseModel):
    """Input schema for the reasoning agent endpoint."""
    question: str
    thread_id: str = "default"
    max_iterations: int = 10


class ReasoningResponse(BaseModel):
    """Output schema."""
    answer: str
    iterations: int
    thread_id: str
    status: str = "success"


async def main(req: func.HttpRequest) -> func.HttpResponse:
    """
    Azure Function: POST /api/reasoning-agent

    Accepts a JSON body with a question, runs the multi-step reasoning agent,
    and returns the answer.

    Request body:
        {"question": "What is the population of Tokyo divided by London?"}

    Response:
        {"answer": "...", "iterations": 4, "thread_id": "default", "status": "success"}
    """
    logging.info("Reasoning Agent function triggered.")

    # Parse and validate request
    try:
        body = req.get_json()
        request = ReasoningRequest(**body)
    except (ValueError, ValidationError) as e:
        return func.HttpResponse(
            json.dumps({"error": f"Invalid request body: {e}"}),
            status_code=400,
            mimetype="application/json",
        )

    # Run the agent
    try:
        agent = get_agent()
        result = await agent.run(
            user_message=request.question,
            thread_id=request.thread_id,
        )

        response = ReasoningResponse(
            answer=result["answer"],
            iterations=result["iterations"],
            thread_id=request.thread_id,
        )

        return func.HttpResponse(
            response.model_dump_json(),
            status_code=200,
            mimetype="application/json",
        )

    except Exception as e:
        logging.error(f"Agent error: {e}", exc_info=True)
        return func.HttpResponse(
            json.dumps({"error": "Agent execution failed", "detail": str(e)}),
            status_code=500,
            mimetype="application/json",
        )
```

## 11. Agentic Coding Patterns with Claude Code

Claude Code enables **Agentic Coding** — you describe intent, and Claude autonomously implements, tests, debugs, and iterates. Here are the key patterns for this project.

### Pattern 1: Scaffold from Requirements

```bash
# In your project root, start Claude Code
claude

# Prompt:
"Read the existing src/agents/reasoning/graph.py. Add a new 'summarize_sources' 
node that runs after all tool calls complete but before the final answer. 
The node should extract citations from tool results and format them as a 
numbered list appended to the answer. Write tests for this new node."
```

Claude Code will: read the file → understand the graph structure → add the node → wire edges → write tests → run tests → fix failures.

### Pattern 2: Debug with Full Context

```bash
# When you hit an error:
claude "I'm getting this error when running the support agent: 
[paste error]. Fix it — the trace shows it fails in the escalation_node 
when ticket is None."
```

### Pattern 3: Iterative Refinement Loop

```bash
# Use Claude Code for TDD (Test-Driven Development)
claude "Write failing tests first for a new 'rate_limiting' middleware 
that limits each customer_id to 10 agent calls per minute using Redis. 
Then implement the middleware to make those tests pass."
```

### Pattern 4: Automated Code Review

```bash
# Before committing:
claude "Review all files in src/agents/ for:
1. Missing error handling in async functions
2. LLM calls without retry logic  
3. Hardcoded values that should be in config.py
4. Missing type annotations
Fix all issues found."
```

### Pattern 5: Add Observability Autonomously

```bash
claude "Add OpenTelemetry tracing to both Azure Functions.
Each function should create a span with attributes:
- agent_type (reasoning|support)
- ticket_id or thread_id
- resolved (bool)
- iteration_count
Export to Azure Application Insights via OTLP."
```

### Claude Code Best Practices for This Project

| Practice | Command / Approach |
|---|---|
| **Start every session with context** | `claude "Read the README and src/ directory tree first"` |
| **Use `/allowed-tools` flag** | `claude --allowed-tools "Bash,Edit,Read"` |
| **Checkpoint before risky changes** | `git commit -m "checkpoint before agent refactor"` then `claude ...` |
| **Specify acceptance criteria** | Always end prompts with "Tests must pass: `pytest tests/ -v`" |
| **Iterative refinement** | Make small focused requests, not one giant prompt |
